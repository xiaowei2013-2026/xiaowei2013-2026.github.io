# 什么才算"做完了"

<Badge type="tip" text="2026-06-26" />

今天和一个朋友聊天的时候突然意识到一个问题：我好像很难定义"做完"。

### 一个 bug 修好了，算做完吗？

算，也不算。修完是修完了，但如果没写测试，如果没想过这个 bug 为什么会出现，如果没顺手清理旁边那坨坏味道的代码 — 好像也可以继续做。

### 一个功能上线了，算做完吗？

功能是上线了，但文档写了没有？通知了相关人员没有？把相关的 TODO 清掉没有？监控配了没有？

### 到底什么时候算做完？

想了半天，我觉得大概是这样的标准：

> **做完 = 你愿意在半年后，深夜里被一个不熟悉这个系统的人拉起来问"这东西怎么跑"，而你不用临时翻代码就能说明白。**

达不到这个标准，就是欠了债。欠的债迟早要还。

但也想开了 — 不是每件事都值得做到这个标准。

有些事就是"跑通了就行"，有些事是"能用就好"，只有少数真正重要的事，才值得那种"彻底做完"的认真。

**分辨什么值得彻底做完，可能比做完更重要。**
